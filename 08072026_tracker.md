## Key Learnings

- `spark-sql` job type on Dataproc doesn't load Iceberg JARs properly (ClassNotFoundException) — use `pyspark` with a script file instead
- Correct BigLake catalog JAR for Dataproc 2.2: `gs://spark-lib/biglake/biglake-catalog-iceberg1.5.1-0.1.2-with-dependencies.jar`
- Cluster name is `lakehouse-cluster` (not `eastside-cluster`)
- BQ external tables for JSONL: field names must match exactly what's in the file
- Deleting Iceberg tables via BQ `bq rm` removes BQ linked dataset but NOT the BLMS catalog entry — must use BLMS API directly
- Ab Initio uses confidence margin (gap between best and 2nd-best) as a signal — worth implementing
